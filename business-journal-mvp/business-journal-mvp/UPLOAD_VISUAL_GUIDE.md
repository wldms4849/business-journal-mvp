# 🎨 그림으로 보는 GitHub 업로드 - 3가지 방법

개발을 모르는 분도 쉽게 따라할 수 있는 **3가지 방법**을 소개합니다!

---

## 방법 1: 🚀 자동 스크립트 사용 (제일 쉬움!)

### 📍 준비물
- GitHub 계정
- Git 설치됨

### 📝 단계

```
1️⃣  GitHub에서 새 저장소 만들기
     ↓
2️⃣  프로젝트 폴더 열기
     ↓
3️⃣  스크립트 실행
     ↓
4️⃣  아이디/이메일 입력
     ↓
5️⃣  완료! ✅
```

### Windows 사용자:
1. `business-journal-mvp` 폴더 열기
2. `upload-to-github.bat` 파일 **더블클릭**
3. 검은 창이 열리면:
   - GitHub 아이디 입력 → Enter
   - GitHub 이메일 입력 → Enter
4. 자동으로 업로드! 🎉

### Mac 사용자:
1. Terminal 열기
2. 다음 명령어 입력:
   ```bash
   cd ~/Desktop/business-journal-mvp
   ./upload-to-github.sh
   ```
3. GitHub 아이디 입력 → Enter
4. GitHub 이메일 입력 → Enter
5. 자동으로 업로드! 🎉

### ⚠️ 주의사항
- 먼저 GitHub에서 빈 저장소를 만들어야 해요!
- 비밀번호 입력 창이 나오면 입력하세요

## 방법 2: 💻 GitHub Desktop 사용 (프로그램)

### 📍 준비물
- GitHub 계정
- GitHub Desktop 설치

### 📝 단계

```
1️⃣  GitHub Desktop 다운로드
    https://desktop.github.com
     ↓
2️⃣  설치 후 GitHub 로그인
     ↓
3️⃣  File → Add Local Repository
     ↓
4️⃣  business-journal-mvp 폴더 선택
     ↓
5️⃣  "Publish repository" 클릭
     ↓
6️⃣  완료! ✅
```

### 상세 설명

#### 1. GitHub Desktop 설치
- https://desktop.github.com 접속
- 운영체제에 맞는 버전 다운로드
- 설치 프로그램 실행

#### 2. 로그인
- GitHub Desktop 실행
- "Sign in to GitHub.com" 클릭
- 웹브라우저에서 승인

#### 3. 저장소 추가
```
┌─────────────────────────┐
│  File → Add Local Repo  │
└─────────────────────────┘
         ↓
┌─────────────────────────┐
│  Choose 버튼 클릭        │
│  business-journal-mvp   │
│  폴더 선택              │
└─────────────────────────┘
         ↓
┌─────────────────────────┐
│  Add repository 클릭     │
└─────────────────────────┘
```

#### 4. 업로드
```
┌─────────────────────────┐
│  왼쪽 하단에           │
│  변경된 파일 목록 보임  │
└─────────────────────────┘
         ↓
┌─────────────────────────┐
│  Summary 입력란에       │
│  "Initial commit" 입력  │
└─────────────────────────┘
         ↓
┌─────────────────────────┐
│  "Commit to main" 클릭  │
└─────────────────────────┘
         ↓
┌─────────────────────────┐
│  상단 "Publish repo" 클릭│
└─────────────────────────┘
         ↓
        완료! 🎉
```

## 방법 3: 🌐 웹에서 직접 업로드 (파일 적을 때)

### 📍 준비물
- GitHub 계정
- 웹브라우저

### 📝 단계

이 방법은 파일이 많아서 **권장하지 않지만**, 작은 프로젝트에는 가능합니다.

```
1️⃣  GitHub에서 새 저장소 만들기
     ↓
2️⃣  "uploading an existing file" 클릭
     ↓
3️⃣  파일/폴더 드래그 앤 드롭
     ↓
4️⃣  "Commit changes" 클릭
     ↓
5️⃣  완료! ✅
```

### ⚠️ 주의
- 파일이 많으면 시간 오래 걸림
- 폴더 구조 깨질 수 있음
- **방법 1 또는 2 추천!**

## 💡 꿀팁

### Tip 1: 비밀번호 대신 토큰 사용
```
GitHub → Settings → Developer settings
→ Personal access tokens → Tokens (classic)
→ Generate new token
→ repo 체크 → 생성
→ 나온 토큰을 비밀번호 대신 사용!
```

### Tip 2: 한 번에 여러 파일 수정 후 업로드
```bash
# 변경사항 확인
git status

# 모두 추가
git add .

# 커밋
git commit -m "수정 내용 설명"

# 업로드
git push
```

### Tip 3: 실수로 잘못 올렸을 때
```bash
# 마지막 커밋 취소
git reset --soft HEAD~1

# 다시 올리기
git add .
git commit -m "올바른 내용"
git push --force
```
